Technical Tasks

Network Data & Design

This task encompasses the development of schedule-based transit networks in both regions, access and transfer links, and any setup or maintenance tools to facilitate the continued use of the tool.

Interim Work Products:
  • Network Data Standard
  • Transit Network Creation – with EMME
  • Transit Network Creation – with Network Wrangler
  • SF-CHAMP Fast-Trips Networks, v1.4
  • Dwell Time Estimations

Transit Demand

This task will validate, test, and adjust disaggregate transit demand data from the two regions using available observed data in order to ensure the demand inputs for Fast-Trips calibration are consistent with observed behavior. This will allow us in the calibration phase to ascertain the network model deficiencies, and not confound them with demand issues.

Interim Work Products:
  • Demand Data Standards
  • Demand Translation – SF-CHAMP
  • Demand v0.2 – SF-CHAMP
  • Demand Translation – SoundCast
  • Demand v0.1 – SoundCast

Transit Rider Behavior

This task includes the data development from the observed transit route choice and the estimation and calibration of route choice models capable of capturing appropriate heterogeneity and sensitivity of transit rider behavior. This task will be informed by SFCTA’s past experience estimating a bicycle route choice model and CTR’s past experience estimating a transit route choice model in Austin.

Interim Work Products:
  • Assignment Results Preparation Script
  • dyno-path data standard for transit paths
  • Route survey calibration data preparation

Transit Performance Data

This task includes the collation, evaluation, and packaging of transit performance data that will be used to evaluate the performance of the modeling system.

Software Implementation

This task develops the transit-simulation software Fast-Trips and includes the refactoring of Fast-Trips to Python, addition of features for skimming, route choice generation, and the development of a performance indicator library.

Interim Work Products:
  • Fast-Trips core software

Test Case Development

This task includes the specifying and management of various scenarios used for testing software validity and model performance.

Implementation, Testing, and Calibration

This task includes the implementation and testing of the Fast-Trips transit simulation into the SF-CHAMP and SoundCast model systems, and the calibration of Fast-Trips in the SF-CHAMP system.
